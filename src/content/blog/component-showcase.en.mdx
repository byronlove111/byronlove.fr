Every article on this blog is an MDX file. That means I can embed React components directly in the prose — not as an afterthought, but as a first-class part of the writing. Here's a tour of everything available.

---

## CodeBlock

For featured code blocks: monospace header plus a copy control — "**copy**" glides lightly upward and fades as "**copied**" appears; it switches back shortly after (~0.9s).

<CodeBlock client:load filename="src/pages/blog/[...slug].astro" lang="astro">
```astro
---
const { slug } = Astro.params;
const post = await getEntry("blog", slug);
const { Content } = await post.render();
---

<article>
  <h1>{post.data.title}</h1>
  <Content />
</article>
```
</CodeBlock>

The filename prop is optional — leave it out for ephemeral snippets. The `lang` prop sets the label in the header when there's no filename.

<CodeBlock client:load lang="bash">
```bash
npm run build
git push origin main
```
</CodeBlock>

## Demo

A bordered wrapper for embedded interactive islands. Anything with `useState` lives here — just add `client:load` to the inner component.

## Native Markdown

Everything you'd expect: **bold**, *italic*, `inline code`, and links.

> Blockquotes for emphasis. Use sparingly — if everything is emphasized, nothing is.

Ordered lists for sequences:

1. Read the conversation history
2. Ask five questions
3. Generate the article

Unordered lists for collections:

- MDX for rich content
- Astro content collections for schema and routing
- Framer Motion for animations

Horizontal rules (`---`) create breathing room between major sections.

---

That's the full toolkit. Each component is designed to disappear — the chrome serves the content, not the other way around.
