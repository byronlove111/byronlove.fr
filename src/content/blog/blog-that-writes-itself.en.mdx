I hate writing. Or more precisely, I hate writing *after* building. The context is still fresh, the insights are right there, but putting them into words feels like running a second marathon after the first one.

So I didn't. I built a system to do it for me.

## The problem with "I'll write about it later"

Every builder I know has a graveyard of things they meant to write about. Not because they're lazy — because the energy required to ship and the energy required to write are fundamentally different. One is flow state. The other is editorial distance.

The insight I had: what if the article was generated *during* the build session, while the context is still loaded?

> The best time to write about what you built is right after you built it. The second best time is never.

## How it works

The whole thing is a Cursor skill — a markdown file that tells the AI exactly how to behave when triggered. You type `write-article` in any chat, it reads the conversation history to understand what you built, asks you five questions, then writes two MDX files: one in English, one in French.

The skill knows to inject the right components — `<CodeBlock>` for snippets, `<Demo>` for interactive bits — based on what you tell it during the Q&A.

## The MDX stack

Each article is an `.mdx` file in `src/content/blog/`. Astro's content collections handle the schema, routing, and type safety. The rendered article page loads components globally so no imports are needed inside the article itself — except for custom demo components, which need `client:load` to hydrate on the client.

The bilingual part was non-negotiable. I work and think in both languages, and if I'm going to write, I want both audiences to get the full thing — not a machine-translated afterthought.

## What I actually got

The first real article I published with this took me twelve minutes from triggering the skill to having it live on my domain. That included answering the questions, reviewing the output, and pushing to GitHub.

> The skill reads the conversation history, not git. So it works even if you haven't committed yet.

The quality surprised me. It writes in my voice — short sentences, no filler, honest about trade-offs — because it calibrates against the existing content on the site before generating anything.

## The only thing left to do

Write more things worth writing about.
